# Applications and Systems

## Sebastian Andres Manotas Blanco

### EJERCICIO 1: Calculadora básica
Razonamiento: En este ejercicio quise aplicar las operaciones aritméticas fundamentales y practicar el uso del switch. El programa pide dos números y un operador, y dependiendo del símbolo realiza la operación. También agregué una validación para evitar dividir entre cero. La idea era entender cómo tomar decisiones y repetir el proceso usando un ciclo.
### EJERCICIO 2: Sistema de reserva de boletos de cine
Razonamiento: Aquí el objetivo fue representar una sala de cine usando una matriz de asientos. Cada posición del arreglo indica si un asiento está disponible o reservado. El programa imprime el mapa, recibe la fila y el asiento que el usuario quiere y actualiza la reserva si está libre. Este ejercicio me ayudó a entender mejor cómo manejar arreglos bidimensionales y validaciones.
### EJERCICIO 3: Sistema de gestión de calificaciones
Razonamiento: Este sistema guarda nombres y calificaciones en arreglos paralelos. Permite agregar estudiantes, verlos, buscar uno por nombre y calcular estadísticas como el promedio o la calificación más alta. El ejercicio me sirvió para practicar el uso de cadenas, menús y recorridos de arreglos, además de organizar la información de manera sencilla.
### EJERCICIO 4: Conversor de monedas
Razonamiento: El programa convierte dólares a diferentes monedas usando tasas de cambio fijas. El usuario elige la opción y el valor se multiplica por la tasa correspondiente. También hay una opción personalizada. La idea principal era trabajar con menús, operaciones matemáticas simples y selección de opciones mediante switch.
### EJERCICIO 5: Sistema De Autenticación 
Razonamiento: Para este ejercicio implementé un pequeño sistema de usuarios usando arreglos para almacenar nombres y contraseñas. Se puede registrar un usuario, iniciar sesión, cambiar la clave o cerrar sesión. La parte central es comparar cadenas con strcmp y usar una variable booleana para saber si hay una sesión activa. Me ayudó a entender cómo manejar datos tipo texto y estados.
### EJERCICIO 6: Evaluador de expresiones
Razonamiento: La idea fue leer una expresión matemética escrita como texto y evaluarla sin usar librerías externas. El programa recorre la cadena, identifica números y operadores y va calculando el resultado. Es un método sencillo, pero fue útil para entender cómo procesar texto caracter por caracter y simular un evaluador básico.
### EJERCICIO 7: Sistema de votación
Razonamiento: Este sistema permite registrar candidatos y luego contar los votos que reciben. Al final, muestra el total de votos y un gráfico de barras hecho con asteriscos, además del ganador. Con este ejercicio practiqué arreglos, ciclos, conteo y porcentajes, además de mostrar información de forma más visual.
### EJERCICIO 8: Sistema de seguimiento de puntajes
Razonamiento: Aquí se manejan jugadores, sus puntajes acumulados y la cantidad de partidas que han jugado. También se puede ordenar la tabla de posiciones de mayor a menor usando un método de burbuja. Me sirvió para entender cómo funcionan los arreglos paralelos y cómo ordenar datos manualmente.
### EJERCICIO 9: Calculadora de calorías y BMI
Razonamiento: En este ejercicio apliqué fórmulas relacionadas con salud: índice de masa corporal, metabolismo basal y calorías quemadas. El programa pide algunos datos del usuario y realiza los cálculos. Esta parte fue buena para practicar operaciones matemáticas un poco más complejas y dividir el programa en funciones más específicas.
